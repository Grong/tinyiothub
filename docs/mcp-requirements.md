# TinyIoTHub MCP 协议支持 - 需求调研

## 一、背景

随着 AI 工具的快速发展，MCP (Model Context Protocol) 已成为连接 AI 应用与外部系统的开放标准。TinyIoTHub 作为云端 SaaS 物联网平台，支持 MCP 协议将使其能够：

1. **AI 助手控制**：通过自然语言查询和控制 IoT 设备
2. **智能自动化**：AI Agent 可调用 IoT 设备能力
3. **数据洞察**：AI 分析 IoT 设备数据并生成报告

## 二、MCP 协议概述

### 2.1 什么是 MCP？

MCP (Model Context Protocol) 是一个开放协议，用于将 AI 应用连接到外部系统。就像 USB-C 为电子设备提供标准连接方式一样，MCP 为 AI 应用提供标准方式来连接外部数据源和工具。

### 2.2 MCP 核心能力

MCP 服务器可以提供三种主要能力：

| 类型 | 说明 | IoT 场景示例 |
|------|------|-------------|
| **Resources** | 可读取的数据（类似文件） | 设备列表、设备状态、传感器数据 |
| **Tools** | 可调用的函数（需用户批准） | 开关设备、设置参数、查询历史数据 |
| **Prompts** | 预定义的提示模板 | 设备故障排查报告生成 |

### 2.3 架构模式

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  AI Agent   │────▶│ MCP Client  │────▶│ MCP Server  │
│ (Claude等)  │     │             │     │ (TinyIoTHub)│
└─────────────┘     └─────────────┘     └─────────────┘
                                                 │
                                                 ▼
                                         ┌─────────────┐
                                         │  IoT 设备   │
                                         │ Modbus/ONVIF│
                                         └─────────────┘
```

### 3.3 技术需求

#### 3.3.1 传输方式

| 方式 | 适用场景 | 复杂度 |
|------|----------|--------|
| **STDIO** | 本地运行 | 简单 |
| **HTTP/SSE** | 远程部署 | 中等 |

**建议**：首期支持 STDIO，后续支持 HTTP

#### 3.3.2 认证方式

- **方案 A**：无认证（仅本地使用）
- **方案 B**：API Key 认证（推荐）
- **方案 C**：OAuth 2.0

**建议**：首期支持 API Key 认证，与现有 JWT 体系兼容

#### 3.3.3 Rust MCP SDK

目前官方提供：
- Python SDK (`mcp`)
- TypeScript/JavaScript SDK (`@modelcontextprotocol/sdk`)

**Rust 生态**：
- `mcp-core` - 核心协议实现
- 社区实现的 Rust SDK

**建议**：评估 `mcp-core` 或参考官方 SDK 自行实现

### 3.4 非功能需求

| 需求 | 目标 |
|------|------|
| 性能 | 工具响应时间 < 500ms |
| 可用性 | 7x24 运行，支持热更新 |
| 安全性 | API Key 加密传输，权限控制 |
| 兼容性 | 支持 Claude Desktop、Cursor 等主流 AI 客户端 |

| 风险 | 影响 | 缓解措施 |
