# 市场功能实施路线图

## 快速开始方案（推荐）

### 方案 A：静态市场（最简单）
**适合场景**：快速上线，资源较少，更新不频繁

**实现方式**：
1. 在项目中创建 `marketplace/` 目录
2. 存放 JSON 格式的资源元数据
3. 前端直接读取 JSON 文件
4. 资源文件托管在 CDN 或 GitHub Releases

**优点**：
- 无需额外服务器
- 实现简单快速
- 维护成本低

**缺点**：
- 无法统计下载量
- 无法实时更新
- 功能有限

### 方案 B：混合市场（推荐）
**适合场景**：中等规模，需要基本统计，可接受定期更新

**实现方式**：
1. 官方资源：静态 JSON + CDN
2. 客户端缓存：本地数据库
3. 定期同步：每天更新一次元数据
4. 下载统计：可选的简单统计服务

**优点**：
- 平衡了复杂度和功能
- 可以逐步扩展
- 成本可控

**缺点**：
- 需要定期维护元数据
- 统计功能有限

### 方案 C：完整市场（最强大）
**适合场景**：大规模部署，需要完整功能，有运维资源

**实现方式**：
1. 独立市场服务器
2. 数据库存储元数据
3. CDN 托管资源文件
4. 完整的统计和分析

**优点**：
- 功能完整
- 可扩展性强
- 用户体验好

**缺点**：
- 开发成本高
- 运维复杂
- 需要服务器资源

## 推荐实施步骤（方案 B）

### 第一步：准备市场数据结构

```
marketplace/
├── index.json              # 市场索引
├── templates/
│   ├── index.json         # 模板列表
│   ├── modbus-sensor.json # 模板详情
│   └── ...
└── drivers/
    ├── index.json         # 驱动列表
    ├── bacnet.json        # 驱动详情
    └── ...
```

### 第二步：实现本地缓存和安装

```rust
// 核心功能
1. 从市场 URL 获取资源列表
2. 缓存到本地数据库
3. 下载资源文件
4. 验证和安装
5. 记录已安装资源
```

### 第三步：创建前端界面

```typescript
// 页面结构
/marketplace/templates     # 模板市场
/marketplace/drivers       # 驱动市场
/marketplace/installed     # 已安装
```

### 第四步：测试和发布

```
1. 准备测试资源
2. 端到端测试
3. 文档编写
4. 发布上线
```

## 需要的决策

1. **选择哪个方案？**
   - 建议：方案 B（混合市场）

2. **资源托管在哪里？**
   - GitHub Releases（免费）
   - 阿里云 OSS / 腾讯云 COS（便宜）
   - 自建 CDN（灵活）

3. **市场数据存放在哪里？**
   - GitHub 仓库（推荐，免费且方便）
   - 独立服务器（更灵活）

4. **更新频率？**
   - 每天自动同步（推荐）
   - 手动触发更新
   - 实时更新
