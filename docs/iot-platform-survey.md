### 4.1 目标用户场景

|------|----------|-------------------|

### 4.2 竞争优势

| 优势 | 说明 | 护城河 |
|------|------|--------|
| **边缘网关** | 本地运行，离线可用 | 核心定位 |

### 4.3 竞争劣势

| 劣势 | 影响 | 弥补方式 |

| 生态弱 | 集成能力不足 | MCP 协议 |

## 七、总结

### 7.1 核心发现

1. **云平台功能完善但重**：AWS/Azure/阿里云功能全但部署重，适合大规模
2. **开源方案灵活但散**：ThingsBoard/Home Assistant 功能全但需要自运维
3. **TinyIoTHub 定位清晰**：边缘网关 + 轻量 + AI 集成，差异化竞争

### 7.2 建议策略

| 方向 | 说明 |
|------|------|
| **聚焦边缘** | 强化离线运行、本地控制 |
| **AI 优先** | MCP 集成做成标杆 |
| **工业突破** | Modbus/ONVIF/PLC 深度支持 |
| **生态合作** | 对接 HA/涂鸦，不重复造轮子 |
