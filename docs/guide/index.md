## 核心概念

### 设备 (Device)

设备是 TinyIoTHub 的核心实体，代表物理或虚拟的 IoT 设备。每个设备具有：

- **唯一标识**: SN 序列号
- **设备类型**: 传感器、执行器、摄像头、网关等
- **驱动类型**: 支持的协议（Modbus、ONVIF、SNMP 等）
- **配置参数**: 连接参数和数据点
- **状态**: 在线/离线/告警

### 驱动 (Driver)

驱动是设备与系统通信的桥梁。TinyIoTHub 支持：

- Modbus RTU/TCP（工业设备）
- ONVIF（视频监控）
- SNMP（网络设备）
- Ping（主机存活检测）
- 自定义驱动（动态加载）

### 告警 (Alarm)

告警系统实时监控设备数据，支持：

- 阈值告警（温度、湿度等超过阈值）
- 离线告警（设备心跳丢失）
- 故障告警（设备主动上报）
- 状态变化告警（开关状态变化）
- 多通道通知（SMS、邮件、Webhook、MQTT）

### 自动化 (Automation)

自动化规则将触发器和动作关联，实现无人值守的自动控制：

- **事件触发**: 设备数据变化时自动执行
- **定时触发**: 按 Cron 表达式定时执行
- **动作类型**: 设备命令、通知、HTTP 请求等

### 模板 (Template)

设备模板简化设备创建过程，预定义：

- 驱动类型
- 配置参数
- 数据点映射
- 告警规则

## 快速入门

### 1. 添加设备

1. 进入「设备管理」→「设备列表」
2. 点击「添加设备」
3. 选择设备模板或手动配置驱动
4. 填写连接参数（IP、端口等）
5. 保存并等待设备上线

### 2. 配置告警规则

1. 进入「告警管理」→「告警规则」
2. 点击「添加规则」
3. 选择要监控的设备和数据点
4. 设置告警阈值和通知方式
5. 启用规则

### 3. 创建自动化

1. 进入「自动化」→「规则列表」
2. 点击「添加规则」
3. 配置触发条件和执行动作
4. 设置冷却时间和优先级
5. 启用规则

### 4. 邀请团队成员

1. 进入「用户管理」
2. 点击「添加用户」
3. 填写用户信息和角色
4. 用户使用初始密码登录后修改

## 高级功能

- **驱动动态加载**: 在运行时加载和卸载驱动
- **设备追踪**: 记录设备所有操作和通信
- **定时任务**: 定时执行 HTTP 请求、脚本或 SQL
- **市场**: 从市场安装驱动和模板
- **多租户**: 支持多个独立租户
