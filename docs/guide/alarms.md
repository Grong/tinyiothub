# 告警管理

告警系统实时监控设备状态，及时发现和处理异常情况。

## 告警类型

| 类型 | 说明 |
|------|------|
| 阈值告警 | 数据超过设定阈值 |
| 离线告警 | 设备断开连接 |
| 故障告警 | 设备报告故障 |
| 自定义 | 用户自定义规则 |

## 创建告警规则

1. 进入「告警管理」页面
2. 点击「添加规则」
3. 选择告警类型
4. 配置触发条件
5. 设置通知方式
6. 保存规则

### 阈值告警配置

```json
{
  "device_id": "device_001",
  "data_point": "temperature",
  "condition": ">",
  "value": 30,
  "duration": 60
}
```

## 告警级别

| 级别 | 颜色 | 说明 |
|------|------|------|
| 紧急 | 🔴 | 需要立即处理 |
| 重要 | 🟠 | 需要尽快处理 |
| 一般 | 🟡 | 需要关注 |
| 信息 | 🔵 | 提示信息 |

## 通知方式

### MQTT 推送

告警信息通过 MQTT 实时推送：

```
gateway/{sn}/alarm
```

### 邮件通知

配置 SMTP 后，告警可发送邮件通知。

## 告警处理

### 确认告警

收到告警后，点击「确认」标记已处理。

### 告警记录

所有告警历史记录可查询和导出。
