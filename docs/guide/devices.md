# 设备管理

设备管理是 TinyIoTHub 的核心功能，用于统一管理所有接入的物联网设备，包括传感器、执行器、网关和各种智能设备。

## 添加设备

### 方式一：手动添加

1. 登录管理后台
2. 进入「设备管理」页面
3. 点击「添加设备」按钮
4. 填写设备基本信息：
   - **设备名称**：具有辨识度的名称，如"一楼温度传感器"
   - **设备序列号 (SN)**：设备的唯一标识符
   - **设备类型**：传感器、执行器、摄像头、网关等
   - **驱动类型**：选择设备使用的通信协议
   - **连接地址**：设备的网络地址或串口参数
5. 点击「保存」

### 方式二：使用设备模板

1. 进入「设备模板」页面
2. 选择合适的模板（如 Modbus TCP 温湿度传感器）
3. 点击「使用模板创建设备」
4. 根据向导填写配置参数（IP、端口、从机 ID 等）
5. 确认创建设备

### 方式三：网关自动发现

某些驱动支持网关自动发现设备，进入「网关管理」页面查看已发现的设备列表。

## 设备状态

| 状态 | 标识 | 说明 |
|------|------|------|
| 在线 | 🟢 | 设备正常连接，数据正常上报 |
| 离线 | 🔴 | 设备断开连接或心跳丢失 |
| 告警 | 🟡 | 设备触发告警规则 |
| 未知 | ⚪ | 尚未收到设备数据 |

**在线状态判断规则：**
- 设备在 10 分钟内有数据更新视为在线
- 设备属性在 5 分钟内有更新视为在线属性
- 设备心跳超时（默认 30 秒）判定为离线

## 设备操作

### 查看详情

点击设备列表中的设备卡片或名称，可查看：
- **基本信息**：名称、序列号、类型、驱动、连接地址
- **实时数据**：所有属性的当前值和更新时间
- **历史数据**：属性值随时间变化的趋势图
- **事件记录**：设备产生的告警和事件历史
- **追踪日志**：设备通信和操作追踪记录

### 编辑设备

支持在线编辑设备配置，修改后：
- 连接参数变更需要设备重新连接后生效
- 属性配置变更立即生效

### 启用/禁用设备

- **禁用设备**：设备数据停止采集，但仍保留配置
- **启用设备**：恢复数据采集

### 删除设备

> ⚠️ **警告**：删除设备会同时删除该设备的所有属性数据、历史记录和事件数据，此操作不可恢复！

1. 在设备详情页点击「删除」
2. 确认删除操作
3. 设备及其数据将被永久删除

### 远程控制

根据驱动支持情况，可对设备执行：
- **参数读取**：主动读取设备当前参数值
- **命令下发**：向设备发送控制指令（如开/关、设置参数）
- **固件更新**：通过驱动支持更新设备固件

## 数据采集

设备支持多种数据采集模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 主动上报 | 设备主动推送数据到系统 | 支持主动上报的智能设备 |
| 定时轮询 | 系统按固定间隔读取设备数据 | Modbus、SNMP 等协议设备 |
| 变化触发 | 设备数据变化时采集 | 需要实时监控状态变化的场景 |
| 命令触发 | 由用户手动触发数据采集 | 按需采集场景 |

### 配置定时轮询

```json
{
  "poll_interval_ms": 5000,
  "timeout_ms": 3000,
  "retry_count": 3
}
```

## 设备属性

每个设备可以有多个属性（Property），代表设备的监控数据点：

| 属性 | 温度 | 湿度 | 开关状态 |
|------|------|------|----------|
| 当前值 | 25.5°C | 60.2%RH | ON |
| 数据类型 | float | float | boolean |
| 只读 | ✓ | ✓ | ✗ |
| 最后更新 | 10:30:00 | 10:30:00 | 10:25:00 |

### 读取属性历史

```http
GET /api/v1/devices/{id}/properties/{property_id}/history
```

### 更新可写属性

```http
PUT /api/v1/devices/{id}/properties/{property_id}
Content-Type: application/json

{
  "value": "26.0"
}
```

## 设备指令

设备指令（Command）用于向设备下发控制命令：

### 执行设备指令

```http
POST /api/v1/devices/{id}/commands/{command_id}/execute
Content-Type: application/json

{
  "parameters": {
    "target": 25
  }
}
```

### 获取指令执行历史

```http
GET /api/v1/devices/{id}/command-executions
```

## 设备分组与标签

### 设备分组

将设备按物理位置、功能或其他维度进行分组：

- 按楼层：一楼、二楼、三楼
- 按车间：装配车间、测试车间、仓库
- 按类型：温湿度类、开关类、仪表类

### 标签管理

为设备添加标签便于筛选：

```
标签示例：floor-1, production, temperature, critical
```

## 设备配置文件

设备配置文件（Profile）是设备的完整快照，包含：

```json
{
  "device": { /* 设备基本信息 */ },
  "is_online": true,
  "properties": [ /* 属性列表 */ ],
  "commands": [ /* 指令列表 */ ],
  "statistics": {
    "total_properties": 12,
    "online_properties": 10,
    "total_events": 25,
    "active_alarms": 2
  },
  "generated_at": "2024-01-07 16:00:00"
}
```

获取设备配置：
```http
GET /api/v1/devices/{id}/profile
```

## 设备追踪日志

设备追踪（Trace）记录设备的所有操作和通信：

| 追踪类型 | 说明 |
|----------|------|
| operation | 用户或系统执行的操作 |
| debug | 调试详细信息 |
| performance | 性能指标数据 |
| error | 错误和异常记录 |
| communication | 设备通信协议交互 |

```http
GET /api/v1/devices/{id}/traces
```

## 常见问题

**Q：设备显示离线但实际在线？**
- 检查设备心跳间隔设置
- 确认网络连通性
- 检查防火墙是否阻断了端口

**Q：数据采集不上来？**
- 确认驱动配置正确（地址、端口、从机 ID）
- 检查设备是否支持该驱动协议
- 查看驱动日志排查通信问题

**Q：如何批量管理设备？**
- 使用标签筛选批量选择设备
- 通过模板批量创建设备
