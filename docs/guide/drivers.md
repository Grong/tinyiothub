# 驱动管理

驱动是 TinyIoTHub 与各种设备通信的桥梁。

## 支持的驱动

### 内置驱动

| 驱动 | 协议 | 说明 |
|------|------|------|
| Modbus RTU | 串口 | 支持 RTU 模式 |
| Modbus TCP | 网络 | 支持 TCP 模式 |
| ONVIF | 网络 | 视频监控设备 |
| SNMP | 网络 | 网络设备监控 |
| Ping | 网络 | 主机存活检测 |

### 自定义驱动

支持开发自定义驱动，详见 [驱动开发](/drivers/)

## 驱动配置

### Modbus 配置

```json
{
  "protocol": "tcp",
  "host": "192.168.1.100",
  "port": 502,
  "slave_id": 1,
  "timeout_ms": 5000
}
```

### ONVIF 配置

```json
{
  "host": "192.168.1.200",
  "port": 80,
  "username": "admin",
  "password": "password"
}
```

## 驱动状态

| 状态 | 说明 |
|------|------|
| 🟢 已加载 | 驱动正常工作 |
| 🔴 加载失败 | 驱动初始化失败 |
| 🟡 重试中 | 连接失败，正在重试 |

## 管理操作

### 安装驱动

将驱动文件放入 `api/drivers/` 目录，系统自动加载。

### 配置驱动

每个驱动有独立的配置参数，详见驱动文档。

### 查看日志

驱动运行日志可在「系统日志」中查看。
