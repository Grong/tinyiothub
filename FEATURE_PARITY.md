## 三、硬件产品（已选型，二次开发中）

### 产品形态

桌面智能机器人 / 家庭 AI 伴侣

### 硬件能力

| 功能 | 说明 | 优先级 |
|------|------|--------|
| WiFi 设备发现 | 扫描局域网内智能设备 | P0 |
| 设备自动接入 | 一键将设备接入云端 | P0 |
| 网关功能 | MQTT 桥接、云端透传 | P0 |
| 本地 AI 对话 | 内置 AI 语音助手 | P0 |
| 蓝牙 BLE 扫描 | 发现蓝牙设备 | P1 |
| 红外控制 | 控制空调/电视等红外设备 | P1 |
| 边缘计算 | 本地规则执行 | P2 |

### 设备发现流程

```
1. 硬件开机，连接 WiFi
2. 扫描局域网设备（MDNS/SSDP）
3. 识别设备类型（通过协议指纹）
4. 自动匹配设备模板
5. 一键接入 TinyIoTHub 云端
6. 完成！用户可以开始使用
```

## 五、用户体验

### 用户路径

```
1. 购买硬件设备
   ↓
2. 开机，连接 WiFi
   ↓
3. 扫码绑定账号
   ↓
4. 硬件自动扫描并发现设备
   ↓
5. 设备自动接入云端
   ↓
6. 对着硬件说："帮我关灯"
   ↓
7. AI 理解指令 → 调用 TinyIoTHub API
   ↓
8. 控制设备 → 完成
```

### 核心优势

| 对比传统 IoT 平台 | TinyIoTHub |
|------------------|-------------|
| 用户需要配置物模型（20+项） | 选择模板（1步） |
| 需要懂技术才能接入 | 硬件自动发现 |
| API 文档复杂难懂 | MCP 简单调用 |
| 每个设备单独控制 | AI 统一管理 |
